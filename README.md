# 発問データセット
発問データセットは解答の自動評価手法を検討するために、学習者の解答とそれに対する評価ラベルを付与したデータセットになります。

発問データセットでは高等学校の授業科目である「情報1」に関する発問を、文部科学省で公開されている[教員研修用教材](https://www.mext.go.jp/a_menu/shotou/zyouhou/detail/\\1416756.htm)から120問作成し、それに対する解答を450個収集しました。

また、3人の評価者に各解答の評価を依頼し、得られた評価ラベルから各解答の最終的な評価ラベルを決定しました。

# 評価ラベル

本データセットでは正解か不正解かの単純な評価ではなく、より詳細な評価ラベルを学習者の解答に付与しています。付与する評価ラベルは７つであり、以下に各ラベルの説明を示します。

|評価|ラベル|説明|
|:---:|:---:|:---|
ドメイン外|1|情報Iに関する内容が解答に全く含まれていない場合に「ドメイン外」と評価しました。また「わかりません」や「答えられません」などの解答にもドメイン外と評価しました。ドメイン外に対するラベルは「1」とします。
無関係|2|情報Iに関する内容が解答に含まれているが、発問の内容を全く含んでいない場合に「無関係」と評価しました。無関係に対するラベルは「2」とします。
不十分|3|発問に求められる型と解答の型が一致していない場合、「不十分」と評価しました。また発問の内容に沿った解答であるが、模範解答が求める解答の内容を十分に満たしていない場合に「不十分」と評価しました。不十分に対するラベルは「3」とします。
矛盾|4|発問の内容に沿った解答であるが、語句の関係が明らかに間違っている、あるいは発問の内容と矛盾した解答である、列挙した語句が全て正しくない場合に「矛盾」と評価しました。矛盾に対するラベルは「4」とします。
欠如|5|語句の列挙を求める発問 (~ が挙げられますか、~にはどのような種類がありますか) に対して、正しい語句が1つ以上含まれているが、1つあるいは複数が不足している、あるいは一部が正しくない場合に「欠如」と評価しました。なお、列挙されているものが全て正しくない場合は欠如と評価せず、「矛盾」と評価しました。欠如に対するラベルは「5」とします。
異なる修飾|6|模範解答が求める語やそれらの正しい関係を解答例が含んでいるが、それらを修飾している語、句、節が1つ以上正しくない、あるいは1つ以上不足している場合に「異なる修飾」と評価しました。異なる修飾に対するラベルは「6」とします。
正解|7|模範解答と解答例が完全に一致、あるいは言い換えである場合に「正解」と評価しました。模範解答が求める内容より多くの内容が解答例に含まれていたとしても、模範解答の内容を解答例が満たしていれば、正解としました。その際、模範解答が求める内容以上のことに関する正誤は考慮していません。正解に対するラベルは「7」とします。

これらの評価ラベルに従って、3人の評価者にそれぞれ450個の解答例に対するアノテーションを依頼しました。3人の評価者が付与した評価ラベルに対して、Fleiss' kappa係数を計算し、評価者間でラベルが適度に一致していることを確認しました。
```bash
評価者の数 = 3
評価対象の数 = 450
評価カテゴリー数 = 7
kappa = 0.432653488509699
適度に一致
```

得られたアノテーション結果から、2人以上の評価者が一致した評価したラベルを、その解答例に対する最終的な評価として決定しました。また3人の評価者がそれぞれ異なる評価ラベルを付与した場合、3人のうち最も低いラベルをその解答例に対する評価ラベルとしました。データセットに含まれる各評価ラベルの個数を以下に示します。

|評価|ラベル|個数|
|:---:|:---:|:---:|
ドメイン外|1|2個
無関係|2|14個
不十分|3|112個
矛盾|4|13個
欠如|5|38個
異なる修飾|6|90個
正解|7|181個

# Test of unseen answers (TUA)
Test of unseen answers (TUA)では学習データに含まれない解答例に対する評価精度を検証するテスト形式です。本リポジトリのHatsumonDataGeneratorではTUAをクロスバリデーション(fold = 5)で行うためにデータセットを返す関数が用意されています。

各発問あたり3〜5個の解答例が存在します。各発問あたり１つの解答例をテストデータとし、残りの解答例を学習データとして使用します。

TUAを行う時の学習データサイズとテストデータサイズを以下に示します。

|TUA|train data size|test data size|
|:---:|:---:|:---:|
|| 330| 120|

# Test of unseen questions (TUQ)
Test of unseen questions (TUQ)では学習データに含まれない発問の解答例に対する評価精度を検証するテスト形式です。本リポジトリのHatsumonDataGeneratorではTUQをクロスバリデーション(fold = 5)で行うためにデータセットを返す関数が用意されています。

各学習あたり5つの発問が存在し、各発問あたり3〜5個の解答例が存在します。、各学習の1つの発問に対する解答例をテストデータ、残りの発問に対する解答例を学習データとして使用します

TUQを行う時の学習データサイズとテストデータサイズを以下に示します。

|TUQ|train data size|test data size|
|:---:|:---:|:---:|
|| 360| 90|
